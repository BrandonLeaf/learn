## 软件评价

+ 核心
    + 可维护性
    + 可复用性
    > **要做到可维护的复用** (设计原则为指导原则)

+ 维护困难的原因
    + 过于僵硬
    + 过于脆弱
    + 复用率低
    + 粘度过高

+ 设计目标
    + 可扩展性
    + 灵活性
    + 可插入性

## 设计原则

### 开-闭原则

1. 什么是开闭原则
    + 对扩展开放
    + 对修改关闭

1. 优越性
    1. 允许扩展已有软件系统，适应新需求
    2. 已有软件模块不在修改，特别是重要的抽象层模块不再修改

1. 怎么做
    + 抽象化是关键
        + java接口(interface)
            
        + 抽象类(abstract)
